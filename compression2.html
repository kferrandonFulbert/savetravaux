Compression du code

Si vous avez beaucoup de code html, js, css ou tout se qui agit coter navigateur, il faut compresser votre code (comment ca compresser?). un espace ou un retour a la ligne coute beaucoup d'octet sur une page meme si cela ne parait pas, en enlevant tout ces caractères inutle on améliore les temps de performance d'affichage meme si cela peut paraitre dérisoire. Oui mais la téléphonie prenant de plus en plus de place dans le surf sur internet et que meme en 3G les chargements de site peuvent entre long parter du principe de qui peu le plus peu le moins.

1.1. Le bon compromis de compression

Si vous enregistrez votre image en png ou jpg votre image sera compresser avec le mode par défaut proposer par votre logiciel d'image. Sachez que lorsque vous enregistrez une image dans un format de compression comme jpg ou png vous pouvez définir le taux de compression, a vous de trouver un bon compromis entre un taux de compression fort et une bonne qualité de l'image. Cependant il existe des outils qui permettent de choisir un bon ratio a votre place, ainsi vous pouvez utilisez des outils comme YSLOW proposer par Yahoo ou encore page speed proposé par Google. 1.2. Miniature

Un bon moyen d'ajouter des liens interne à votre site et d'accélérer le temps d'affichage de ces pages consiste à utiliser des miniatures d'images avec un lien vers l'images tailles réelle sans compression. Cette méthode a un avantage souvent négliger, elle vous permettra d'avoir votre image référencer dans les moteurs d'image. Ex: Code source:
<a href="http://abys-music.com/PICTURES/template/equipage/abys.jpg" title="groupe ABYS" target="_blank" > <img src=" http://www.frannuaire-gratuit.com/img/abys.jpg " alt="groupe abys " /> </a>

2. Limiter les requêtes au serveur

Noter bien que lorsque vous faites appelle a une page HTML, PHP, ASP ou autre votre navigateur fait une première requête au serveur pour savoir si la page appeler existe bien et en charger le contenu en cas de valeur retour 200, mais cela ne s'arrête pas là, tous les éléments externe de la page (css, js, image, et autre composant externe comme les outil d'anayse, ou de MAP... etc) demande pour chacun d'eux une requête serveur.

2.1. Les Sprites

Les sprites est une solution qui est bien connu du monde du jeu vidéo. elle consiste a avoir toutes les images de la page dans une seul est même image, il faut ensuite découper l'image pour en utiliser que la parti voulu. Cela permet de limiter les requêtes aux serveurs. Cette solution nécessite certaines connaissance en css et en html . Pour allez plus loin un exemple de sprite css.
